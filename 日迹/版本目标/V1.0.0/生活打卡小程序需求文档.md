## 1. 版本目标

建立可独立使用的账号与习惯打卡能力。用户能够完成“登录 -> 创建打卡计划 -> 查看今日计划 -> 完成或取消打卡 -> 管理个人资料”的基础流程，并确保不同账号间的数据隔离。

### 1.1 上线标准

- 用户可使用微信或邮箱完成注册、登录和找回密码。
- 用户可创建、查看、完成、取消和删除自己的打卡项目。
- 今日完成数、总数和完成率与实际记录一致。
- 用户可编辑头像、昵称、成长号和个人简介。
- 核心数据写入服务端，重新登录或更换设备后仍可恢复。

### 1.2 本版不包含

日记、图片发布、日历回顾、公开社区、互动消息、关注关系、成长统计、订阅提醒、草稿和账号合并均不在本版验收范围。

## 2. 目标用户与场景

- 首次使用者希望快速建立第一个运动、学习或作息计划。
- 日常使用者打开小程序后，希望在两次点击内完成当天打卡。
- 用户误操作后，希望能够取消当天打卡并得到正确统计。
- 用户希望修改个人资料，并在下次登录后继续使用原有数据。

## 3. 信息架构


## 4. 核心流程

### 4.1 首次使用

1. 用户进入登录选择页并勾选用户协议与隐私政策。
2. 用户选择微信登录或邮箱登录。
3. 登录成功后进入打卡页；没有项目时展示空状态和“新建打卡”按钮。
4. 用户填写名称、图标和重复频率后保存。
5. 新项目按频率决定是否进入今日计划。

### 4.2 每日打卡

1. 用户进入今日计划。
2. 点击项目右侧状态按钮完成打卡。
3. 页面即时更新项目状态、完成数、总数、完成率和进度条。
4. 再次点击可取消当天打卡，统计同步回退。

### 4.3 删除项目

1. 用户左滑项目，露出删除操作。
2. 点击删除后展示项目名称和不可恢复提示。
3. 确认后删除项目并重新计算今日统计；取消则恢复项目行。

## 5. 功能需求

### 5.1 账号与登录

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-ACC-001 | P0 | 微信登录 | 用户同意协议后建立微信会话；头像和昵称由用户主动填写或授权 |
| FR-ACC-002 | P0 | 邮箱密码登录 | 校验邮箱格式；密码至少 8 位且同时包含字母和数字 |
| FR-ACC-003 | P0 | 邮箱验证码 | 验证码为 6 位数字，10 分钟有效，60 秒内不可重复发送 |
| FR-ACC-004 | P0 | 邮箱注册 | 未注册邮箱验证成功后设置密码并创建账号 |
| FR-ACC-005 | P0 | 找回密码 | 已注册邮箱验证成功后允许设置新密码，旧会话失效 |
| FR-ACC-006 | P0 | 登录状态 | 有效会话自动登录；失效后回到登录页，禁止丢失本地未提交操作 |
| FR-ACC-009 | P0 | 退出登录 | 清除本机凭证并回到登录页，不删除服务端业务数据 |

安全规则：验证码连续输错 5 次后失效；密码连续失败 5 次后暂停登录 15 分钟；验证码按邮箱、设备和 IP 限流。日志不得记录密码、验证码和完整会话凭证。

### 5.2 打卡项目

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-CHK-001 | P0 | 新建打卡 | 名称 1-20 字；图标和频率必选；保存前实时预览 |
| FR-CHK-002 | P0 | 快速打卡 | 每个项目每天最多一条有效完成记录；重复请求返回相同结果 |
| FR-CHK-003 | P0 | 今日概览 | 展示完成数、项目总数、整数完成率和进度条，状态变化后即时更新 |
| FR-CHK-008 | P0 | 左滑删除 | 左滑露出删除按钮；二次确认后删除并重新计算统计 |
| FR-CHK-009 | P0 | 今日计划 | 仅展示当天应执行且未删除的项目；完成项使用选中和划线状态 |
| FR-CHK-010 | P0 | 空状态 | 没有今日项目时展示明确空状态和新建入口，不显示 0/0 的误导性完成率 |

重复频率首版支持“每天、工作日、周末、每周指定日期”。以服务端配置时区计算有效日期，客户端修改系统时间不得生成重复记录。

删除采用逻辑删除，历史记录保留在服务端但不再展示；V1.0.0 不提供恢复入口。

### 5.3 个人资料与设置

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PRO-001 | P0 | 编辑资料 | 支持头像、昵称、成长号和个人简介 |
| FR-PRO-006 | P0 | 账号设置 | 展示当前登录方式，支持退出登录和查看协议 |
| FR-PRO-007 | P0 | 成长号 | 4-20 位字母、数字或下划线，全局唯一，冲突时不可保存 |

头像仅支持 JPG、PNG、WebP，单张不超过 5 MB；昵称 1-20 字；个人简介最多 80 字。保存失败时保留当前填写内容。

## 6. 数据要求

| 数据对象 | 必要字段 |
| --- | --- |
| 用户 | 用户 ID、登录标识、昵称、头像、成长号、简介、状态、创建时间 |
| 打卡项目 | 项目 ID、用户 ID、名称、图标、频率规则、时区、状态、创建时间 |
| 打卡记录 | 记录 ID、项目 ID、用户 ID、打卡日期、完成状态、完成时间 |

- 所有业务查询必须按当前用户 ID 隔离。
- 同一项目与同一打卡日期只能存在一条有效记录。
- 写操作必须返回明确成功状态；客户端仅在服务端确认后展示最终成功。
- 时间统一存储为 UTC，按用户时区计算“今天”。

## 7. 异常与状态

- 网络失败：保留表单内容并提供重试，不重复创建项目或记录。
- 会话过期：引导重新登录，登录后返回原操作入口。
- 项目已删除：其他设备再次操作时提示“项目不存在或已删除”并刷新列表。
- 成长号冲突：在输入项下方显示原因，不清空其他资料。
- 列表加载：必须包含加载中、加载失败、空数据和正常数据四种状态。

## 8. 验收标准

- [ ] 未同意协议时，微信和邮箱登录均不可执行，并给出提示。
- [ ] 新邮箱可完成验证码注册，已注册邮箱可使用密码或验证码登录。
- [ ] 名称为空、超过 20 字或频率未选择时不能创建打卡项目。
- [ ] 打卡与取消打卡后，项目状态和今日统计立即且正确变化。
- [ ] 连续点击或弱网重试不会产生同日重复打卡记录。
- [ ] 左滑删除必须经过二次确认，取消删除后项目保持不变。
- [ ] 资料校验失败时不丢失已填写内容，保存成功后重新登录仍可读取。
- [ ] 不同账号之间不能读取或修改对方的项目、记录和资料。
- [ ] 主流程在当前正式版微信及前两个主要版本中可正常使用。

## 9. 发布检查

- 账号、项目和记录表已建立唯一约束与索引。
- 隐私政策覆盖微信身份、邮箱、头像和打卡数据的处理方式。
- 开发、测试、生产环境隔离，生产日志已完成敏感字段脱敏。
- P0 验收项全部通过，无阻断级和严重级缺陷。
