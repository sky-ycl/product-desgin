## 1. 版本目标

在 V1.0.0 账号和打卡数据之上增加个人日记、图文发布、关联打卡和日历回顾，完成“创建计划 -> 打卡 -> 写日记 -> 日历回顾”的个人记录闭环。

### 1.1 上线标准

- 用户可发布、查看、编辑和删除仅自己可见的图文日记。
- 用户可将一条日记关联到一个自己的打卡项目。
- 日历可准确标记有日记或打卡记录的日期，并展示当日记录。
- 日记、日历、打卡作为同一页面顶部 Tab，可点击或左右滑动切换。
- 发布页返回时回到进入发布页之前的界面，未提交内容按规则处理。

### 1.2 本版不包含

公开发布、关注可见、发现内容流、点赞、评论、收藏、关注、消息、内容举报、草稿箱、搜索筛选和成长统计不在本版范围。

## 2. 页面与导航

底部导航本版为“日记、发布、我”。尚未开放的首页发现和消息入口隐藏。日记页标题栏固定为“日记、日历、打卡”，默认进入“日记”，同一会话内返回时保留最后选择的 Tab。

| 页面 | 主要内容 | 核心交互 |
| --- | --- | --- |
| 日记 Tab | 个人日记时间线、可见状态、编辑与删除 | 点击查看；管理自己的日记 |
| 日历 Tab | 月历、月份摘要、选中日期记录 | 切月、回到今天、选择日期 |
| 打卡 Tab | V1.0.0 今日计划与新建打卡 | 完成、取消、左滑删除、新建 |
| 发布 | 标题、正文、图片、关联打卡 | 保存日记、返回上一界面 |

顶部 Tab 顺序不可调整为二级小 Tab。内容区域支持水平滑动，滑动阈值应避免与打卡项目左滑删除产生冲突。

## 3. 核心流程

### 3.1 发布日记

1. 用户从底部发布按钮或打卡完成后的记录入口进入发布页。
2. 填写可选标题、必填正文，上传最多 9 张图片。
3. 可选择一个未删除的个人打卡项目；从打卡入口进入时自动带入该项目。
4. 点击发布后保存为“仅自己可见”的日记。
5. 成功后进入日记 Tab 并将新内容置顶；失败时保留编辑内容。

### 3.2 日历回顾

1. 用户切换到日历 Tab，默认选中今天。
2. 有日记或完成打卡的日期显示记录标记。
3. 点击日期后展示该日的日记和打卡结果。
4. 用户可前后切换月份，点击右上角“今天”回到当前日期。

## 4. 功能需求

### 4.1 日记管理

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-DIA-001 | P0 | 日记时间线 | 按发布时间倒序展示当前用户日记，首次加载 20 条并支持继续加载 |
| FR-DIA-002 | P0 | 可见状态 | 本版所有日记固定为“仅自己”，列表和详情均明确展示 |
| FR-DIA-003 | P0 | 编辑与删除 | 仅作者可编辑或删除；删除前二次确认，成功后列表与日历同步更新 |
| FR-DIA-004 | P0 | 日历回顾 | 月视图标记有记录的日期，支持切月、回到今天和查看当日记录 |
| FR-DIA-006 | P0 | 日记详情 | 展示标题、正文、图片、关联打卡和发布时间，不展示社交互动区 |

### 4.2 图文发布

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PUB-001 | P0 | 图文编辑 | 标题选填且不超过 30 字；正文必填且为 1-2000 字 |
| FR-PUB-002 | P0 | 图片上传 | 支持 0-9 张 JPG、PNG、WebP，单张不超过 10 MB |
| FR-PUB-003 | P0 | 关联打卡 | 每条日记最多关联一个当前用户的未删除项目，保存项目名称快照 |
| FR-PUB-007 | P0 | 发布结果 | 保存成功进入日记 Tab；失败时保留本地编辑内容并允许重试 |
| FR-PUB-008 | P0 | 返回上一界面 | 左上角返回箭头回到进入发布页前的界面 |
| FR-PUB-009 | P0 | 离开确认 | 有未发布内容时返回，提示“继续编辑”或“放弃内容” |

图片上传成功后才允许提交日记；用户删除待上传图片后不再纳入提交。图片按选择顺序展示，本版不要求拖动排序。

### 4.3 顶部 Tab 与日历

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-NAV-001 | P0 | 顶部 Tab | 顺序固定为日记、日历、打卡；激活项有明确视觉状态 |
| FR-NAV-002 | P0 | 滑动切换 | 内容区域左右滑动切换相邻 Tab，不循环切换 |
| FR-CAL-001 | P0 | 月历标记 | 任一日存在日记或有效打卡记录时显示标记 |
| FR-CAL-002 | P0 | 日期详情 | 按日记、打卡两类展示选中日期的记录，空日期显示空状态 |
| FR-CAL-003 | P0 | 今天 | 仅日历 Tab 顶部右侧显示“今天”，点击后定位当前年月日 |

## 5. 数据要求

| 数据对象 | 必要字段 |
| --- | --- |
| 日记 | 日记 ID、用户 ID、标题、正文、可见范围、发布时间、更新时间、删除状态 |
| 日记图片 | 图片 ID、日记 ID、文件地址、顺序、宽高、上传状态 |
| 日记打卡关联 | 日记 ID、项目 ID、项目名称快照、关联时间 |

- 标题和正文按纯文本处理，输出时转义脚本和 HTML 标签。
- 日记删除采用逻辑删除，删除后不得继续通过详情地址访问。
- 日历查询按月份返回日期摘要，日期详情按需加载，避免一次返回整月正文和图片。
- 用户仅可关联和读取自己的打卡项目及记录。

## 6. 异常与状态

- 图片上传失败：标记具体失败图片，允许单张重试，不清空正文。
- 发布重复点击：按钮进入提交中状态，服务端使用请求标识避免重复日记。
- 关联项目已删除：允许日记保留项目名称快照，编辑时提示重新选择或取消关联。
- 月历加载失败：保留当前月份，展示重试入口，不错误显示为空月份。
- 跨日停留：重新进入页面或回到前台时刷新“今天”和今日打卡计划。

## 7. 验收标准

- [ ] 日记、日历、打卡直接位于顶部标题栏，点击和左右滑动均可切换。
- [ ] 左右滑动到首尾 Tab 后不能继续循环，且不误触打卡项目删除操作。
- [ ] 标题超过 30 字、正文为空或超过 2000 字时不能发布。
- [ ] 支持上传最多 9 张合规图片，失败后正文和成功图片不会丢失。
- [ ] 发布成功后新日记位于时间线顶部，重新登录后仍可查看。
- [ ] 有未发布内容时点击返回箭头必须提示确认；无内容时直接返回上一界面。
- [ ] 日历正确标记有日记或打卡记录的日期，并显示选中日期详情。
- [ ] 删除日记后，时间线、日历标记和日期详情保持一致。
- [ ] 其他账号无法通过接口或页面访问当前用户的私密日记和图片。

## 8. 发布检查

- 已完成 V1.0.0 数据升级兼容，既有打卡项目和记录不受影响。
- 私密图片使用受控访问地址，不允许公开枚举或永久裸链访问。
- 关键流程覆盖弱网、重复提交、跨日和多设备场景。
- P0 验收项全部通过，无私密内容泄露风险。
