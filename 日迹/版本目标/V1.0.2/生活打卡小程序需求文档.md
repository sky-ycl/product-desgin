## 1. 版本目标

在个人记录闭环稳定后，引入可控的公开分享与轻社区能力。用户可主动将日记公开，在发现页浏览内容，并完成点赞、评论、收藏、关注、举报和消息查看。

### 1.1 上线标准

- 私密日记默认规则保持不变，公开必须由用户主动选择。
- 公开内容经安全审核后才进入发现流和公开主页。
- 点赞、评论、收藏和关注状态在多端保持一致。
- 举报、拉黑和删除能及时阻断不应继续展示的内容。
- 互动事件生成消息，并有明确未读状态。

### 1.2 本版不包含

私信、短视频、直播、排行榜、组队挑战、复杂推荐算法、付费能力和 AI 内容生成不在本版范围。

## 2. 页面范围

底部导航升级为“首页、日记、发布、消息、我”。首页展示发现内容流；日记页继续承载个人日记、日历和打卡；“我”增加公开主页、收藏和关注关系入口。

| 页面 | 主要内容 | 核心交互 |
| --- | --- | --- |
| 首页·发现 | 双列公开内容流 | 加载、刷新、打开详情 |
| 内容详情 | 作者、图文、打卡标签、评论区 | 点赞、评论、收藏、关注、举报 |
| 发布 | V1.0.1 编辑能力、可见范围 | 选择仅自己、关注我的人或公开 |
| 消息 | 互动和系统通知 | 分类查看、单条已读、全部已读 |
| 公开主页 | 公开资料与公开日记 | 关注、取消关注、举报或拉黑 |

## 3. 功能需求

### 3.1 公开发布与发现

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PUB-004 | P0 | 可见范围 | 每次新建默认仅自己，可改为关注我的人或公开，不继承上次选择 |
| FR-PUB-006 | P0 | 内容审核 | 公开内容提交文本和图片安全检查；审核通过后进入发现流 |
| FR-DIS-001 | P0 | 公开内容流 | 双列展示审核通过且未删除的公开日记，支持下拉刷新和分页 |
| FR-DIS-002 | P0 | 内容详情 | 展示作者、正文、图片、关联打卡、发布时间及互动数据 |
| FR-DIS-009 | P0 | 内容失效 | 删除、转私密、审核下架或作者注销后，从发现和详情同步移除 |

发现流首版按发布时间、内容完整度和基础互动加权排序，不采集超出业务需要的用户画像，不建设个性化推荐模型。

### 3.2 社区互动

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-DIS-003 | P0 | 点赞 | 同一用户对同一内容仅一个有效点赞，可取消 |
| FR-DIS-004 | P0 | 评论与回复 | 评论 1-500 字；支持一级评论、回复指定用户和评论点赞 |
| FR-DIS-005 | P0 | 收藏 | 收藏进入“我-收藏”；原内容失效后显示失效状态 |
| FR-DIS-006 | P0 | 关注 | 支持关注与取消关注，不允许关注自己 |
| FR-DIS-008 | P0 | 举报与拉黑 | 举报必须选择原因；拉黑后双方内容和互动按规则隔离 |

作者可删除自己内容下的评论，评论发布者可删除自己的评论。删除评论时同步处理回复展示和计数；本版不支持多级嵌套回复。

### 3.3 消息

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-MSG-001 | P0 | 互动通知 | 生成点赞、收藏、评论、回复和新增关注通知 |
| FR-MSG-002 | P0 | 系统通知 | 展示内容审核、举报处理和账号安全通知 |
| FR-MSG-003 | P0 | 未读状态 | 底部导航显示未读红点，支持单条已读和全部已读 |
| FR-MSG-004 | P0 | 分类查看 | 支持互动、关注和系统三类筛选 |

同一用户对同一内容频繁点赞和取消不重复轰炸作者；聚合策略为同类事件在短时间内合并展示。

### 3.4 个人主页与关系

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PRO-002 | P0 | 公开主页 | 仅展示公开资料、公开日记和公开统计，不展示私密内容 |
| FR-PRO-003 | P0 | 收藏列表 | 查看、打开和取消收藏，失效内容不可进入详情 |
| FR-PRO-005 | P0 | 关注关系 | 查看关注与粉丝列表，支持取消关注和移除粉丝 |
| FR-PRO-008 | P0 | 黑名单 | 查看并解除拉黑；解除后不自动恢复关注关系 |

## 4. 内容安全与隐私

- 私密日记和“关注我的人”日记不得进入公开发现、公开搜索或陌生人主页。
- 改为公开前明确提示内容将进入发现流和个人公开主页。
- 公开转私密后，发现、收藏详情、分享入口和缓存应及时失效。
- 举报原因至少包含违法违规、色情低俗、骚扰攻击、虚假内容、侵权和其他。
- 审核拒绝需说明原因并提供申诉入口；申诉过程记录操作人和时间。
- 拉黑后双方不能关注、评论或查看受限内容，历史互动按产品规则隐藏。

## 5. 数据与一致性

| 数据对象 | 核心约束 |
| --- | --- |
| 点赞/收藏 | 用户 ID 与内容 ID 组合唯一，取消采用状态变更或幂等删除 |
| 评论 | 记录内容、作者、目标内容、被回复用户、状态和时间 |
| 关注 | 关注者与被关注者组合唯一，禁止二者相同 |
| 消息 | 接收者、触发者、事件类型、业务对象、已读状态和时间 |
| 举报/拉黑 | 发起者、目标、原因、处理状态和审计时间 |

- 互动计数以服务端数据为准，异步更新失败时可重新汇总修复。
- 删除或转私密属于高优先级失效事件，应主动清理缓存。
- 内容详情接口必须再次校验可见范围，不能只依赖列表入口控制。

## 6. 异常与状态

- 内容审核中：作者可在个人日记看到审核状态，其他用户不可见。
- 内容已失效：从收藏或消息进入时展示统一失效页，不暴露原正文和图片。
- 互动失败：恢复按钮原状态并允许重试，不虚增计数。
- 评论发送中：禁止重复提交；失败后保留输入内容。
- 被拉黑或权限变化：刷新详情并停止后续互动请求。

## 7. 验收标准

- [ ] 新建日记始终默认仅自己可见，不继承上一条公开状态。
- [ ] 仅审核通过的公开日记会进入发现流和公开主页。
- [ ] 点赞、收藏、关注可取消，重复请求不会产生重复关系或错误计数。
- [ ] 评论长度、回复关系、删除权限和计数符合业务规则。
- [ ] 公开内容转私密或删除后，所有公共入口均不能继续访问。
- [ ] 私密和关注可见内容不能通过接口越权读取。
- [ ] 举报和拉黑立即生效，并保留可审计的处理记录。
- [ ] 消息红点、分类、单条已读和全部已读状态准确。
- [ ] 社区能力异常不会阻塞用户使用日记与打卡基础功能。

## 8. 发布检查

- 内容审核服务、举报后台和应急下架能力已上线。
- 已完成公开内容、评论、昵称和图片的安全测试。
- 已验证删除、转私密、拉黑后的缓存失效和越权保护。
- P0 验收项全部通过，无高危内容安全或隐私问题。
