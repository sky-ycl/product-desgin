## 1. 版本目标

增强长期记录的效率和反馈，在不改变核心导航和默认私密原则的前提下，补充提醒、连续统计、成长数据、搜索筛选、草稿和账号增强能力。

### 1.1 上线标准

- 用户可为打卡项目设置合规的微信订阅提醒。
- 连续天数、累计次数和完成率可解释、可复算。
- 用户可通过筛选和搜索快速找到自己的记录或公开内容。
- 发布中断时可保存和恢复草稿。
- 用户可管理登录方式，并在明确确认后合并重复账号。

### 1.2 本版不包含

付费会员、排行榜、陌生人私信、短视频、直播、商城、AI 总结、自动医疗或心理判断不在本版范围。

## 2. 功能需求

### 2.1 打卡增强

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-CHK-004 | P0 | 连续天数 | 展示当前连续、最长连续和累计完成次数，按项目有效日期计算 |
| FR-CHK-005 | P0 | 编辑与归档 | 可修改名称、频率和提醒；归档后不进入今日计划但保留历史 |
| FR-CHK-006 | P0 | 订阅提醒 | 使用微信订阅消息，先获得用户授权；未授权时不能声称提醒已开启 |
| FR-CHK-007 | P0 | 关联日记入口 | 完成打卡后可进入发布页并自动关联当前项目 |
| FR-CHK-011 | P0 | 统计重算 | 取消打卡、修改频率或时区变化后按统一规则重新计算统计 |

连续规则：仅项目计划执行日参与连续判断；未来日期不计入；当天尚未完成不提前中断连续；删除单条记录或取消当天打卡后立即重算。

### 2.2 搜索与筛选

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-DIA-005 | P0 | 个人记录筛选 | 支持按打卡项目、可见范围和时间范围筛选日记 |
| FR-DIS-007 | P0 | 公开内容搜索 | 支持标题、正文关键词、打卡标签和用户昵称搜索 |
| FR-SEA-001 | P0 | 搜索历史 | 仅保存在当前账号，可单条删除或全部清空 |
| FR-SEA-002 | P0 | 结果权限 | 搜索结果必须再次应用内容可见、拉黑和审核状态规则 |

搜索关键词去除首尾空格后长度为 1-50 字；无结果时展示空状态和清除条件操作。首版按相关度和发布时间排序，不提供复杂个性化排序。

### 2.3 草稿

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PUB-005 | P0 | 手动草稿 | 用户可主动保存标题、正文、图片、关联打卡和可见范围 |
| FR-PUB-010 | P0 | 离开保存 | 有未发布修改时可选择保存草稿、放弃或继续编辑 |
| FR-PUB-011 | P0 | 草稿恢复 | 草稿列表按更新时间倒序，支持继续编辑和删除 |

草稿属于私密数据，不参与审核、搜索、统计或日历标记。草稿关联项目被删除时保留名称快照并提示重新选择。

### 2.4 成长数据

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-PRO-004 | P0 | 成长数据 | 展示日记数、累计打卡、整体完成率、当前和最长连续天数 |
| FR-STA-001 | P0 | 时间范围 | 支持近 7 天、近 30 天和自然月，不允许不同口径混用 |
| FR-STA-002 | P0 | 数据说明 | 每项指标可查看计算口径，数据为空时不展示虚假趋势 |

统计结果允许异步生成，但页面必须显示数据更新时间。用户删除日记或打卡记录后，统计需在可接受时间内更新。

### 2.5 账号增强

| 编号 | 优先级 | 功能 | 业务规则 |
| --- | --- | --- | --- |
| FR-ACC-007 | P0 | 账号绑定 | 支持绑定或解绑微信、邮箱；操作前再次验证，至少保留一种登录方式 |
| FR-ACC-008 | P0 | 账号合并 | 展示两个账号的数据摘要和冲突处理方式，用户确认后执行 |
| FR-ACC-010 | P0 | 账号注销 | 二次验证并说明影响；按法规完成冻结、撤回和数据处理流程 |

账号合并必须具备幂等性和审计记录。日记、图片、项目、记录、社交关系及消息按明确策略迁移；成长号冲突时由用户选择新成长号，不允许静默覆盖。

## 3. 指标口径

| 指标 | 计算方式 |
| --- | --- |
| 累计打卡 | 当前用户所有有效完成记录数量 |
| 计划次数 | 统计区间内，各未删除或当时有效项目应执行次数之和 |
| 整体完成率 | 有效完成次数 / 计划次数；计划次数为 0 时显示“暂无数据” |
| 当前连续 | 截至今天，按项目有效执行日向前连续完成的次数 |
| 最长连续 | 项目历史上连续完成有效执行日的最大次数 |
| 日记数 | 当前用户未删除且已发布的日记数量，不含草稿 |

## 4. 非功能要求

- 搜索首屏在常规网络下目标 2 秒内返回，超时提供重试。
- 成长数据允许缓存，但个人写操作后应触发失效或增量更新。
- 提醒授权状态、计划状态和微信侧发送结果分别记录，便于问题排查。
- 账号合并和注销属于高风险操作，必须记录审计日志并支持失败回滚。
- 新增能力不得降低 V1.0.0、V1.0.1 核心记录流程的可用性。

## 5. 异常与状态

- 提醒未授权或授权失效：明确显示未开启，并提供重新授权入口。
- 搜索服务不可用：不影响日记、打卡和发布主流程。
- 统计生成中：展示上次有效数据及更新时间，不显示全零数据冒充结果。
- 草稿图片失效：保留文字并标记失效图片，允许删除或重新上传。
- 账号合并部分失败：整体回滚或进入可恢复状态，禁止产生半迁移账号。

## 6. 验收标准

- [ ] 连续天数在每天、工作日、周末和指定星期频率下均计算正确。
- [ ] 未获得微信授权时，界面不会将提醒展示为已开启。
- [ ] 编辑或归档项目后，今日计划、日历和统计结果符合统一规则。
- [ ] 个人筛选和公开搜索结果不会绕过可见范围、审核或拉黑限制。
- [ ] 草稿可保存、恢复和删除，且不会出现在日记、日历或搜索中。
- [ ] 成长数据的数值、时间范围和计算说明一致，空数据状态正确。
- [ ] 账号解绑后始终保留至少一种可用登录方式。
- [ ] 账号合并不会丢失日记、图片、打卡记录或社交关系，重复请求结果一致。
- [ ] 增强功能故障时，基础登录、打卡、日记和发布流程仍可使用。

## 7. 发布检查

- 已使用历史数据回放验证连续天数和完成率算法。
- 已验证提醒授权、拒绝、过期及微信发送失败场景。
- 已完成搜索越权测试、草稿隐私测试和账号合并演练。
- 已建立统计重算、合并失败恢复和异常监控机制。
- P0 验收项全部通过，无阻断级和严重级缺陷。
